Tamamono 1.8.x Audit

Browser Storage Security Audit

这份审计只覆盖前端浏览器侧的 localStorage / sessionStorage 使用面，目标是把真正可能带来用户负担、敏感残留、 或跨会话暴露的问题优先收口。

5 已收口的高关注存储链
0 剩余中风险内容载荷面
0 仍留在 localStorage 的令牌类密钥

已完成收口

Closed

工作流草稿不再持久化输入文本

kyuubiki.workbench.workflowDrafts.v1 现在只保存图结构和模板链偏好， 不再把 inputArtifactTexts 落进浏览器本地存储。旧记录会在读取时自动清洗。

Closed

工作流快照不再持久化输入文本

workflowSnapshots 的 payload 已经去敏，只保留图结构；快照指纹也不再把 用户输入文本纳入计算，避免这部分再次进入持久化链路。

Closed

本地工作流库不再持久化输入文本

kyuubiki.workbench.workflowLibrary.v1 保留本地图定义与元数据，但不再长期 保存用户输入文本内容。

Closed

脚本编辑器草稿已从 localStorage 迁到 sessionStorage

kyuubiki-workbench-python-panel 与 kyuubiki-workbench-dsl-panel 现在只保留在会话级存储中。 旧的浏览器本地草稿会在读取时自动迁移并从 localStorage 删除。

Closed

宏预设与 snippet 预设已拦截疑似密钥字段

保存预设时，如果参数键名或宏 payload 字段路径中出现明显像 token、secret、apiKey、 authorization、cookie 这类字段， 当前版本会直接拒绝写盘，避免把疑似凭据长期保留在浏览器本地。

剩余重点

Operational

预设仍然会保留非敏感自动化参数

kyuubiki-workbench-macro-presets 与 kyuubiki-workbench-snippet-presets 当前会保存宏步骤、snippet 参数和项目级预设。 现在疑似密钥字段已经被写盘拦截，但 URL、选择器、业务字段、项目路径等 非密钥自动化参数仍然会作为项目预设长期保留。

  • 建议后续增加“临时预设”和“长期预设”分层。
  • 建议在 UI 上补一条可见提示，明确说明哪些字段不会被允许保存。

低风险保留项

存储项 当前判断 原因
kyuubiki-workbench-settings Low 已做 settings / secrets 分离，令牌类字段只进内存，不再长期落在 localStorage。
sessionStorage 活动日志与安全审计 Low 会话级、短期、不可跨浏览器重启继承，主要是可观察性数据。
sessionStorage 脚本编辑器草稿 Low Python / DSL 草稿仍可自动恢复，但不再跨浏览器重启长期残留。
宏预设与 snippet 预设 Low 疑似密钥字段已在保存时被拒绝，剩余持久化内容主要是非敏感自动化参数。
工作流模板链收藏、算子搜索收藏、最近使用记录 Low 属于偏好类元数据，不直接承载敏感业务内容。
语言包与模板链导入库 Low 主要是结构化配置和文案覆盖，不是运行时秘密。
工作流包维护日志 Low 内容偏向扫描 / 修复回执，属于可清理运行痕迹，不是高敏载荷。

建议落地顺序

  • 第一步：在 UI 上补充预设保存失败时的显式敏感字段提示。
  • 第二步：在系统存储面板里把“内容载荷类存储”和“偏好类存储”分开显示。
  • 第三步：把“本地不会长期保留敏感输入”的策略同步到 UI 文案和用户文档。
  • 第四步：如果后续支持账号体系，再把预设类数据切到显式导出 / 导入或云端托管。
