Installer Remote Deployment · Roadmap

From SSH wrapper to deployment service

Installer remote deployment currently sits at pilot maturity: bounded SSH transport with policy validation, remote node registry records, certificate material preparation, and agent start commands. The target is an idempotent deployment service with explicit plans, durable journals, artifact delivery, host trust, and remote test coverage.

Current state

The current Installer path is useful for controlled LAN and lab pilots. It validates host and workspace policy, wraps SSH and SCP, can bootstrap a workspace command, can start an orchestrated or offline-mesh Rust agent, and can attach certificate material. It is not yet a mature production deployment service because it lacks a durable deployment plan, remote journal, artifact pull flow, rollback, host-key pinning policy, and full integration matrix.

Principles

  • Installer owns deployment and repair, not modeling workflow semantics.
  • SSH is transport; deployment intent must be represented as a plan.
  • Remote actions must be policy-bounded, auditable, and replay-safe.
  • Remote hosts should pull staged artifacts instead of relying on ad-hoc shell state.

Roadmap stages

  1. Policy-bounded SSH transport: implemented; keep host, workspace, URL, identity, and snapshot validation visible.
  2. Structured deployment plan: started; model bootstrap, package sync, verify, start, health-check, and rollback as explicit steps.
  3. Durable remote journal: started; every action emits local and remote records with retry-safe status.
  4. Installer-managed artifact delivery: planned; remote hosts pull staged packages and verify checksums/component integrity.
  5. Host trust hardening: planned; move beyond accept-new for non-dev deployments and bind node identity.
  6. Remote deployment test matrix: planned; combine containerized SSH fixtures with physical lab-host runs.

Operator commands

Use cargo run -p kyuubiki-installer -- remote-deployment-roadmap to print the source-of-truth roadmap from the Installer crate. The GUI and docs should keep reflecting that model rather than diverging into separate maturity claims.

Use cargo run -p kyuubiki-installer -- remote-deployment-plan to preview the first structured remote-agent deployment step contract: policy check, workspace bootstrap, artifact sync, integrity verify, agent start, health check, and rollback guidance.

Use cargo run -p kyuubiki-installer -- remote-deployment-journal to preview the retry-safe journal records that will later be written locally and remotely as each deployment step executes.
