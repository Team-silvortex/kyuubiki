Kyuubiki Workbench UI Automation Contract

This contract defines the stable product-owned DOM anchors for the built-in workbench shell. It exists so wasm Python automation and headless control tooling can target fixed UI surfaces without depending on translated copy, visual styling, or user-customized layout.

Contract version: 1 Product-owned shell: yes User-extensible UI shell: no

Rules

  • The workbench shell structure is product-owned and not user-extensible.
  • Automation must target stable data-* selectors from this contract instead of visual text.
  • Accessible labels used for automation are product-owned ids, not localized copy.
  • Built-in sidebar, inspector, console, viewport, and runtime panels remain product-owned surfaces.
  • Workflow, operator, and dataset extensions may change content, but not the shell selector contract.
  • Breaking changes to these selectors require a contract version bump.

Intended Consumers

  • Built-in wasm Python automation running in the frontend.
  • Headless SDK layers that need cross-surface UI assertions.
  • Regression checks for fixed shell layout and control-plane navigation.
