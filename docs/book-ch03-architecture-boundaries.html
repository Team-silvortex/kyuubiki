The hard architecture boundaries

Kyuubiki depends on preserving splits between frontend, Hub, Workbench, Installer, orchestrator, and agents. The agent is not the UI, the Workbench is not the runtime, and the Hub is not the orchestrator.

Back to book Previous chapter Next chapter

Boundary rules

  • Hub manages entry, operator flow, runtime switching, and packaging posture.
  • Workbench owns the concrete analysis workflow and authoring surface.
  • Installer owns deployment structure and installation policy.
  • Agents stay runtime-side, headless, and Rust-first.
  • One orchestrator controls an agent at a time unless the system is intentionally in decentralized mesh mode.

Why these boundaries exist

Once roles blur, automation, deployment, and distributed execution all get harder to reason about. The system becomes easier to extend when ownership and authority stay visible.
