Reading paths by job

Different readers should enter through different doors. These paths keep orientation practical for operators, engineers, release owners, and large-model integrators.

Back to book Previous chapter

Role-based reading

  • Operator / product owner: current line -> system overview -> app runtime boundaries -> operations -> security.
  • Frontend / Workbench engineer: philosophy -> frontend style -> frontend implementation -> UI automation contract -> rendering roadmap.
  • Runtime / agent engineer: system overview -> agent-orchestrator boundary -> headless agent contract -> protocols -> operations.
  • SDK / extension engineer: operator SDK -> headless SDKs -> workflow graph -> workflow dataset -> protocols.
  • Release / packaging owner: packaging and deployment -> desktop release checklist -> installation integrity contract -> update catalog.
  • Large model integrator: book -> book manifest -> system overview -> app runtime boundaries -> protocols -> headless SDKs -> operator SDK.
