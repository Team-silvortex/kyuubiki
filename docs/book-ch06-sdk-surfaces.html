SDK and automation surfaces

There are multiple extension surfaces, and they must not be confused. Runtime extension, headless automation, and frontend-owned UI automation solve different problems and need different contracts.

Back to book Previous chapter Next chapter

Surface split

  • Rust operator SDK: authoring operator capabilities and extending the solver/runtime side.
  • Headless SDKs: automation, agents, external tools, and AI clients outside the frontend.
  • WASM Python UI automation: frontend-owned and stability-sensitive, built around product-owned selector ids instead of localized copy.

Design rule

Mixing these surfaces makes boundaries fuzzy and introduces accidental coupling. Keeping them distinct lets the project harden both frontend automation and runtime automation safely.
