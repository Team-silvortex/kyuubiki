What Kyuubiki is

Kyuubiki is an engine-first FEM workstation and control plane. It is not one monolithic app. It is a coordinated system split across desktop shells, a browser workbench, an orchestrator, and Rust solver agents.

Back to book Next chapter

Role map

  • Hub: operator shell, runtime control, workload shelf, packaging and docs entrypoint.
  • Workbench: modeling, analysis setup, results review, and future deeper 3D authoring.
  • Installer: bootstrap, deployment prep, install integrity, and cleanup contract.
  • Orchestrator: job control, persistence, workflow coordination, remote/runtime management.
  • Rust agents: solver data plane, headless execution, and peer-networking evolution.

Why this split matters

The product needs room to scale from local single-machine use into distributed execution without fusing UI and runtime responsibilities. Treating each role as separate keeps contracts visible and lets the system evolve without destabilizing the whole stack.
