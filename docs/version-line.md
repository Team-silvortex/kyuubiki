# Version Line: tamamono 1.x

`tamamono 1.x` is the current formal Kyuubiki version line.

It marks the point where the project stops treating version movement mainly as
feature expansion and starts treating it as product hardening:

- stronger numerical trust
- smoother operator experience
- broader regression coverage
- more consistent desktop and browser behavior

## What this line means

- the codename for this generation is `tamamono`
- the line opens at the formal product version `tamamono 1.0.0`
- unless there is a special structural reset later, the major version is
  expected to remain `1`

That means future releases should usually look like:

- `tamamono 1.1.0`
- `tamamono 1.2.0`
- `tamamono 1.2.1`

## Relationship to the 0.9.0 release pack

The `0.9.0` release documents remain important. They are the preserved record
of the initial usable release baseline:

- support matrix
- study coverage
- smoke matrix
- first-run path
- troubleshooting note
- readiness and decision notes

The `1.x` line builds on that evidence rather than replacing it.

## Current emphasis

For `tamamono 1.x`, the priority order is:

1. numerical accuracy and benchmark trust
2. bug fixing and consistency
3. operator and modeling experience
4. selective, high-value capability expansion only when it strengthens the line
