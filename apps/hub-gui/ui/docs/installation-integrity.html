Visible behavior contract

Every repair-side behavior should be operator-visible. Some rules are intentionally read-only, but none of them should be hidden.

cleanup

removable residue allowlist

Only these known cache and stale runtime artifacts are eligible for automatic cleanup.

Value: .DS_Store, Thumbs.db, ._*, tmp/run/*.pid, tmp/run/*.sock, tmp/run/*.lock, tmp/run/*.tmp

Mode: Read-only

layout

platform release root

Platform-scoped release staging root tracked by the installer.

Value: dist/{platform}

Mode: Read-only

layout

runtime roots

Standard repo-local roots that every platform install must expose.

Value: tmp/, dist/, runtimes/, releases/

Mode: Read-only

runtime

embedded runtime manifest

Installer-managed Elixir/OTP and Node payloads are visible in the staged release manifest before launch.

Value: dist/{platform}/manifests/embedded-runtimes.json

Mode: Read-only

protection

protected paths

Repair must not remove user environment, runtime data, release snapshots, or desktop shells.

Value: .env.local, tmp/data, releases/snapshots, apps/hub-gui, apps/workbench-gui, apps/installer-gui

Mode: Read-only

version

required shipping version

Desktop shells, workspace packages, and local release descriptors are expected to align to this working version.

Value: 1.11.5

Mode: Read-only

workflow

repair action

Repair is deterministic and intentionally narrow so it never becomes a hidden cleanup burden.

Value: prepare standard layout, then remove allowlisted residue

Mode: Read-only

Required install layout

  • workspace root: . (required)
  • runtime temp root: tmp (required)
  • runtime process state: tmp/run (required)
  • runtime data state: tmp/data (required)
  • release staging root: dist (required)
  • installer-managed runtime payloads: runtimes (required)
  • platform release staging: dist/{platform} (required)
  • release snapshots: releases/snapshots (required)
  • desktop installer shell: apps/installer-gui (required)
  • desktop hub shell: apps/hub-gui (required)
  • desktop workbench shell: apps/workbench-gui (required)

Protected paths

  • .env.local
  • tmp/data
  • releases/snapshots
  • apps/installer-gui
  • apps/hub-gui
  • apps/workbench-gui
