Main deployment modes

  • Local workstation: frontend + orchestrator + local Rust agents with SQLite.
  • Cloud control plane: frontend and orchestrator deployed centrally with PostgreSQL.
  • Distributed control plane: remote solver agents discovered by manifest or registry.
  • Direct mesh / offline mesh: selective mode where direct agent paths and mesh authority matter as much as orchestrated paths.

Agent discovery modes

  • Static: explicit local agent endpoints.
  • Manifest: runtime loads remote agents from a manifest file.
  • Registry: agents register, heartbeat, and unregister dynamically.

Operator entrypoints

  • Use Hub for desktop readiness, runtime control, and guided packaging flows.
  • Use Workbench for focused modeling, analysis, and result inspection.
  • Use Installer for heavier bootstrap and deployment setup.

Authority and mesh posture

  • Keep orchestrated and direct-mesh modes explicit instead of blending them into one hidden runtime state.
  • Installer-owned remote control handles node bootstrap, certificate alignment, mesh preparation, and workflow snapshot inspection.
  • One orchestrator controls an agent at a time unless the node is intentionally operating in offline mesh mode.
