Database profile

Choose your deployment mode

Local first

SQLite workstation

Embedded database with minimal host burden. Best for single-machine work and fast setup.

  • No separate database service
  • Best default for local design iteration
  • Writes to a repo-local `.sqlite3` file

Cloud ready

PostgreSQL deployment

Recommended for multi-node workers, cloud services, and shared environments.

  • Distributed orchestration friendly
  • Stable for shared projects and history
  • Requires `DATABASE_URL`

Control plane

Distributed orchestrator

Keep frontend and orchestrator centralized while remote Rust agents self-register into the cluster.

  • PostgreSQL-backed control plane
  • Remote solver nodes register and heartbeat in
  • Installer can bootstrap and start remote agents over SSH

Service control

Run the whole stack

Local stack

Starts the frontend, orchestrator, and agents with SQLite forced.

Cloud stack

Starts the same services with PostgreSQL forced for shared or distributed deployment.

Distributed control plane

Starts only the frontend and orchestrator, leaving solver capacity to remote registered agents.

Stop services

Stops frontend, orchestrator, and all Rust agents without touching your data.
